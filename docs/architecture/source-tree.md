# Source Tree

The following directory structure reflects the monorepo architecture, component organization, and file-based storage design:

```plaintext
CAUST/                                    # Monorepo root (https://github.com/vios-s/CAUST)
├── README.md                             # Project documentation
├── pyproject.toml                        # Poetry-style project metadata
├── requirements.txt                      # Pinned Python dependencies (NFR11)
├── setup.py                              # Package setup for local development
├── .gitignore
├── .gitmodules                           # Git submodules config
│
├── aust/                                 # Core application package and runtime assets
│   ├── configs/                          # Configuration files (YAML)
│   │   ├── prompts/                      # Agent prompt configurations
│   │   ├── thresholds/                   # Evaluation thresholds
│   │   ├── tasks/                        # Seed templates per task type
│   │   └── personas/                     # Judge personas and roles
│   ├── experiments/                      # Experiment manifests and results
│   ├── logs/                             # Structured logs emitted by scripts
│   ├── outputs/                          # Persistent per-task artifacts (loop state, reports, debates)
│   ├── rag_paper_db/                     # Paper metadata + FAISS index artifacts
│   ├── scripts/                          # CLI utilities and runnable entry points
│   │   ├── main.py
│   │   ├── generate_paper_cards.py
│   │   └── ...
│   ├── src/                              # Application source code
│   │   ├── agents/                       # LLM-powered agent implementations
│   │   ├── loop/                         # Inner/outer loop orchestration + state
│   │   ├── memory/                       # Long-term memory system integration
│   │   ├── rag/                          # Retrieval augmented generation pipeline
│   │   ├── toolkits/                     # Execution adapters (DeepUnlearn, concept erasure, etc.)
│   │   └── logging_config.py             # Central logging setup
│   ├── tests/                            # Project-scoped unit/integration tests
│   └── utils/                            # Shared helpers for scripts and agents
│
├── data/                                 # Models, interaction logs, evaluation assets
├── docker/                               # Dockerfiles and job manifests
├── docs/                                 # Architecture docs, PRDs, stories, guides
├── external/                             # Editable vendored dependencies (CAMEL, DeepUnlearn, ESD)
├── htmlcov/                              # Coverage reports generated by pytest --cov
├── logs/                                 # CLI / orchestration run logs
├── tests/                                # Harness-level tests (smoke / integration shells)
│   └── unit/
├── .bmad-core/                           # BMAD framework metadata
└── .backup, .papers, ...                 # Developer-local caches and scratch dirs
```

**Key Directory Responsibilities:**
- **aust/src/loop/**: Orchestration logic, state machines, and repositories
- **aust/src/agents/**: LLM-powered agent implementations (generator, critic, evaluator, etc.)
- **aust/src/toolkits/**: Execution adapters and DeepUnlearn integration layers
- **aust/src/rag/**: Retrieval system (indexing, retrieval, paper corpus management)
- **aust/src/memory/**: Long-term memory for successful discoveries
- **aust/configs/**: Prompt, threshold, task, and persona configurations
- **aust/scripts/**: CLI entry points for running workflows and utilities
- **aust/outputs/**: Persistent task results (attack traces, reports, debates)
- **external/**: Editable third-party dependencies (CAMEL, DeepUnlearn, ESD)
- **docker/**: Containerization assets and GPU job manifests
