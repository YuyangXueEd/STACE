# Data Models

The following data models represent the core business entities and data structures in AUST. These models are implemented as Pydantic models in the centralized **`aust/src/data_models/`** package for validation, serialization, and type safety.

## Data Models Package Structure

**Location:** `aust/src/data_models/`

The package is organized by functional domain:

```
aust/src/data_models/
├── __init__.py           # Central exports for all models
├── task_spec.py          # TaskSpec
├── hypothesis.py         # Hypothesis, HypothesisContext
├── critic.py             # CriticFeedback
├── debate.py             # DebateExchange, DebateSession
└── loop_state.py         # InnerLoopState, IterationResult, ExitCondition
```

**Benefits:**
- **Single source of truth** - All data models in one location
- **Easy imports** - `from aust.src.data_models import Hypothesis, TaskSpec, ...`
- **Better maintainability** - Changes propagate consistently across agents
- **Type safety** - Pydantic validation ensures data integrity
- **JSON serialization** - Built-in support for persistence

## LoopState

**Purpose:** Tracks the current state of the inner research loop, enabling restart/resume capabilities and state persistence across container restarts.

**Key Attributes:**
- `task_id`: str - Unique identifier for the research task (e.g., "data-unlearning-task-001")
- `task_type`: str - Task type: "data_based_unlearning" or "concept_erasure"
- `current_iteration`: int - Current iteration number (1-10)
- `max_iterations`: int - Maximum allowed iterations (default 10)
- `current_state`: str - State machine state: "HYPOTHESIS_GENERATION", "CRITIC_DEBATE", "RAG_RETRIEVAL", "EXPERIMENT_EXECUTION", "EVALUATION", "FEEDBACK", "COMPLETED", "FAILED"
- `vulnerability_found`: bool - Flag indicating if vulnerability was discovered
- `started_at`: datetime - Loop start timestamp
- `updated_at`: datetime - Last state update timestamp
- `metadata`: dict - Additional task-specific metadata (dataset, model, target)

**Relationships:**
- Has many `IterationResult` (one per iteration)
- Has one `AttackTrace` (aggregated from all iterations)

**Persistence:** Serialized to JSON at `aust/outputs/{task_id}/loop_state.json`, saved after each state transition.

## IterationResult

**Purpose:** Captures the results of a single inner loop iteration, including hypothesis, retrieved papers, experiment results, and evaluation feedback.

**Key Attributes:**
- `iteration_number`: int - Iteration number (1-10)
- `hypothesis`: Hypothesis - Generated hypothesis for this iteration
- `critic_feedback`: str | None - Critic agent's challenge/feedback (None for iteration 1)
- `rag_queries`: list[str] - Search queries generated by Query Generator
- `retrieved_papers`: list[RetrievedPaper] - Papers retrieved from RAG (top-k results)
- `experiment_params`: dict - Parameters passed to experiment executor
- `experiment_results`: ExperimentResult - Results from DeepUnlearn or concept-erasure tool
- `evaluation`: Evaluation - Evaluator's assessment of results
- `feedback_for_next`: str - Feedback message to next iteration
- `timestamp`: datetime - Iteration completion time
- `duration_seconds`: float - Time taken for this iteration

**Relationships:**
- Belongs to one `LoopState`
- Contains one `Hypothesis`, multiple `RetrievedPaper`, one `ExperimentResult`, one `Evaluation`

**Persistence:** Stored in `AttackTrace` (aggregated) and in `aust/outputs/{task_id}/iterations/iteration_{n}.json` for debugging.

## Hypothesis

**Purpose:** Represents a proposed vulnerability test generated by the Hypothesis Generator agent.

**Key Attributes:**
- `hypothesis_id`: str - Unique identifier
- `attack_type`: str - Type of attack (e.g., "membership_inference", "model_inversion", "data_extraction", "concept_leakage")
- `description`: str - Natural language description of the hypothesis
- `rationale`: str - Reasoning for why this attack might succeed
- `target`: str - What the attack targets (e.g., specific data points, concepts)
- `expected_outcome`: str - What success looks like
- `confidence_score`: float - Generator's confidence (0.0-1.0)
- `novelty_score`: float - Estimated novelty based on memory/RAG (0.0-1.0)
- `generated_at`: datetime - Generation timestamp

**Relationships:**
- Belongs to one `IterationResult`

**Persistence:** Embedded in `IterationResult` JSON.

## RetrievedPaper

**Purpose:** Represents a research paper retrieved from the RAG system during a query.

**Key Attributes:**
- `paper_id`: str - Unique identifier (filename hash or DOI)
- `title`: str - Paper title
- `authors`: list[str] - List of authors
- `year`: int - Publication year
- `abstract`: str - Paper abstract
- `retrieved_chunks`: list[str] - Specific chunks/sections retrieved (with context)
- `relevance_score`: float - Similarity score from vector search (0.0-1.0)
- `citation_key`: str - BibTeX citation key for Reporter to use

**Relationships:**
- Referenced by multiple `IterationResult` (many-to-many via retrieval)
- Sourced from `PaperCorpus`

**Persistence:** Metadata stored in vector DB index; full paper stored as PDF in `rag/papers/` directory.

## ExperimentResult

**Purpose:** Contains the raw results from executing an experiment via DeepUnlearn or concept-erasure FunctionTools.

**Key Attributes:**
- `experiment_id`: str - Unique identifier
- `tool_used`: str - "deepunlearn" or "concept_erasure"
- `method`: str - Specific method executed (e.g., "SISA", "gradient_ascent", "EraseDiff")
- `execution_status`: str - "success", "failed", "timeout"
- `metrics`: dict - Method-specific metrics (e.g., {"forget_accuracy": 0.23, "retain_accuracy": 0.89})
- `outputs`: dict - Additional outputs (model checkpoints, generated images, logs)
- `error_message`: str | None - Error details if execution_status == "failed"
- `execution_time_seconds`: float - Time taken for experiment
- `gpu_job_id`: str | None - Kubernetes job ID if applicable

**Relationships:**
- Belongs to one `IterationResult`

**Persistence:** Embedded in `IterationResult` JSON; large artifacts (model checkpoints, images) stored in `aust/outputs/{task_id}/experiments/{experiment_id}/`.

## Evaluation

**Purpose:** Represents the Evaluator agent's assessment of experiment results, determining if a vulnerability was found and providing feedback.

**Key Attributes:**
- `evaluation_id`: str - Unique identifier
- `evaluation_type`: str - "threshold_based" (data-based) or "vlm_based" (concept-erasure)
- `vulnerability_detected`: bool - Whether a vulnerability was found
- `confidence`: float - Evaluator's confidence in the assessment (0.0-1.0)
- `evidence`: str - Explanation of why vulnerability was/wasn't detected
- `threshold_comparison`: dict | None - For threshold-based: {"metric": "forget_accuracy", "threshold": 0.3, "actual": 0.23, "passed": true}
- `vlm_analysis`: dict | None - For VLM-based: {"prompt": "...", "response": "...", "leakage_detected": true}
- `feedback`: str - Actionable feedback for next iteration (or confirmation if vulnerability found)

**Relationships:**
- Belongs to one `IterationResult`

**Persistence:** Embedded in `IterationResult` JSON.

## AttackTrace

**Purpose:** Aggregates the complete research journey from initial hypothesis to vulnerability discovery, formatted for paper integration and user reproduction.

**Key Attributes:**
- `task_id`: str - Links to LoopState
- `task_type`: str - "data_based_unlearning" or "concept_erasure"
- `vulnerability_found`: bool - Overall success flag
- `total_iterations`: int - Number of iterations completed
- `trace_steps`: list[TraceStep] - Ordered list of iteration summaries
- `final_vulnerability`: Hypothesis | None - The successful hypothesis (if found)
- `final_evidence`: str - Summary of evidence supporting the vulnerability
- `reproducibility_instructions`: str - Step-by-step instructions for reproduction
- `created_at`: datetime - Trace creation timestamp

**Relationships:**
- Belongs to one `LoopState`
- Aggregates information from multiple `IterationResult`

**Persistence:** Dual format - JSON at `aust/outputs/{task_id}/attack_trace.json` and Markdown at `aust/outputs/{task_id}/attack_trace.md`.

## TraceStep

**Purpose:** Summarizes a single iteration within an AttackTrace for readability.

**Key Attributes:**
- `iteration`: int - Iteration number
- `hypothesis_description`: str - Brief description of the hypothesis
- `key_papers`: list[str] - Titles of most relevant papers retrieved
- `experiment_summary`: str - What was tested and how
- `result_summary`: str - Outcome of the experiment
- `feedback_summary`: str - What was learned for next iteration

**Relationships:**
- Embedded within `AttackTrace`

**Persistence:** Embedded in `AttackTrace` JSON/Markdown.

## Report

**Purpose:** Represents the academic paper generated by the Reporter agent in the outer loop.

**Key Attributes:**
- `report_id`: str - Unique identifier
- `task_id`: str - Links to LoopState
- `title`: str - Paper title
- `abstract`: str - Paper abstract
- `sections`: dict - Keyed sections: {"introduction": "...", "methods": "...", "experiments": "...", "results": "...", "discussion": "...", "conclusion": "..."}
- `citations`: list[str] - BibTeX citations extracted from retrieved papers
- `figures`: list[str] - Paths to generated figures/visualizations
- `generated_at`: datetime - Report generation timestamp

**Relationships:**
- Belongs to one `LoopState`
- References multiple `RetrievedPaper` (for citations)
- Has many `JudgeEvaluation` (3-5 judges)

**Persistence:** Markdown at `aust/outputs/{task_id}/report.md`, LaTeX at `aust/outputs/{task_id}/report.tex` (optional).

## JudgeEvaluation

**Purpose:** Represents a single LLM judge's evaluation of a Report from a specific perspective.

**Key Attributes:**
- `judge_id`: str - Unique identifier
- `persona`: str - Judge persona: "Security Expert", "ML Researcher", "Privacy Advocate", "Skeptical Reviewer", "Industry Practitioner"
- `evaluation_criteria`: list[str] - Criteria evaluated (novelty, rigor, reproducibility, impact, exploitability)
- `scores`: dict - Scores per criterion: {"novelty": 8, "rigor": 7, "reproducibility": 9, "impact": 7, "exploitability": 8}
- `overall_score`: float - Weighted average score (0-10)
- `strengths`: list[str] - Identified strengths of the research
- `weaknesses`: list[str] - Identified weaknesses or concerns
- `comments`: str - Detailed feedback from judge's perspective
- `recommendation`: str - "Accept", "Minor Revisions", "Major Revisions", "Reject"

**Relationships:**
- Belongs to one `Report`

**Persistence:** JSON at `aust/outputs/{task_id}/judges/judge_{persona}.json`.

## AgentPromptConfig

**Purpose:** Configuration for agent prompts, enabling easy iteration and version control.

**Key Attributes:**
- `agent_name`: str - "hypothesis_generator", "critic", "query_generator", "evaluator", "reporter", "judge"
- `task_type`: str | None - "data_based_unlearning", "concept_erasure", or None for generic
- `system_prompt`: str - Agent's system/role prompt
- `user_prompt_template`: str - Template with placeholders (e.g., "{retrieved_papers}", "{past_results}")
- `model`: str - OpenRouter model to use (e.g., "anthropic/claude-3.5-sonnet")
- `temperature`: float - Sampling temperature (0.0-1.0)
- `max_tokens`: int - Maximum response tokens

**Relationships:**
- Loaded by agents at initialization

**Persistence:** YAML files in `configs/prompts/{agent_name}_{task_type}.yaml`.

## MemoryEntry

**Purpose:** Represents a stored successful vulnerability discovery in CAMEL-AI's long-term memory for future reference.

**Key Attributes:**
- `entry_id`: str - Unique identifier
- `task_type`: str - "data_based_unlearning" or "concept_erasure"
- `successful_hypothesis`: Hypothesis - The hypothesis that discovered a vulnerability
- `key_insights`: list[str] - Lessons learned from this discovery
- `attack_pattern`: str - General pattern that can be reused
- `created_at`: datetime - Memory storage timestamp

**Relationships:**
- Referenced by Hypothesis Generator for inspiration

**Persistence:** Managed by CAMEL-AI memory system (implementation-specific storage).

## New Models (Story 1.5 Implementation)

### DebateSession

**Purpose:** Tracks the complete multi-round debate session for a single inner loop iteration.

**Location:** `aust/src/data_models/debate.py`

**Key Attributes:**
- `iteration_number`: int - Which inner loop iteration this debate belongs to
- `task_id`: str - Task identifier for logging
- `task_type`: str - "data_based_unlearning" or "concept_erasure"
- `debate_enabled`: bool - Whether debate was actually run (False for iteration 1)
- `exchanges`: list[DebateExchange] - All debate rounds in order
- `total_rounds`: int - Number of completed debate rounds
- `quality_threshold_met`: bool - Whether final quality score met threshold (≥0.85)
- `convergence_reached`: bool - Whether improvement stalled (delta < 0.05)
- `quality_threshold`: float - Target quality threshold (default: 0.85)
- `started_at`: datetime - Debate start timestamp
- `completed_at`: datetime - Debate completion timestamp
- `final_hypothesis`: Hypothesis | None - Best hypothesis from debate
- `generator_model`: str - Model used for hypothesis generation
- `critic_model`: str - Model used for critique
- `rag_queries`: list[str] - All RAG queries generated during debate (aggregated)
- `retrieved_papers`: list[dict] - All papers retrieved during debate (aggregated)

**Computed Properties:**
- `duration_seconds`: float - Total debate duration
- `final_quality_score`: float | None - Final critic evaluation score

**Relationships:**
- Belongs to one `IterationResult`
- Contains multiple `DebateExchange` (one per round)
- References final `Hypothesis`

**Persistence:** Saved to `aust/outputs/{task_id}/debates/debate_iteration_{N}.json`

### DebateExchange

**Purpose:** Represents a single round in the multi-round debate between Generator and Critic.

**Location:** `aust/src/data_models/debate.py`

**Key Attributes:**
- `round_number`: int - Which debate round (1, 2, 3, ...)
- `initial_hypothesis`: Hypothesis - Hypothesis at start of this round
- `critic_feedback`: CriticFeedback - Critic's evaluation and suggestions
- `refined_hypothesis`: Hypothesis | None - Improved hypothesis after refinement (None if last round)
- `rag_queries`: list[str] - RAG queries generated this round (from QueryGeneratorAgent)
- `retrieval_context`: list[dict] - Papers retrieved this round for evidence
- `generator_model`: str - Model identifier for generator
- `critic_model`: str - Model identifier for critic
- `timestamp`: datetime - When this exchange occurred
- `improvement_delta`: float | None - Quality improvement from previous round

**Workflow:**
```
Round N:
1. initial_hypothesis provided by generator
2. critic_feedback generated by critic agent
3. rag_queries generated by QueryGeneratorAgent based on feedback
4. retrieval_context populated by RAG system
5. refined_hypothesis generated incorporating evidence and feedback
6. improvement_delta calculated comparing to previous round
```

**Relationships:**
- Belongs to one `DebateSession`
- Contains one initial `Hypothesis`, one `CriticFeedback`, optionally one refined `Hypothesis`

**Persistence:** Part of parent `DebateSession` JSON

### CriticFeedback

**Purpose:** Structured feedback from Critic Agent evaluating hypothesis quality on multiple dimensions.

**Location:** `aust/src/data_models/critic.py`

**Key Attributes:**
- `novelty_score`: float (0.0-1.0) - How novel is this approach vs. past attempts?
- `feasibility_score`: float (0.0-1.0) - Can this be executed with available tools/data?
- `rigor_score`: float (0.0-1.0) - Is the success criterion well-defined and testable?
- `strengths`: list[str] - What aspects of the hypothesis are strong
- `weaknesses`: list[str] - What aspects need improvement
- `suggestions`: list[str] - Specific actionable improvements
- `overall_assessment`: str - Summary assessment paragraph
- `overall_assumption`: str | None - Key assumptions identified in hypothesis

**Computed Properties:**
- `average_score`: float - Mean of (novelty + feasibility + rigor) / 3

**Usage:**
- Generated by Critic Agent after evaluating each hypothesis
- Used by QueryGeneratorAgent to identify knowledge gaps
- Fed back to Generator for next refinement round
- Tracked across debate rounds to measure improvement

**Validation:**
- All scores must be between 0.0 and 1.0
- All list fields must have at least one entry
- overall_assessment must be non-empty string

**Persistence:** Part of `DebateExchange` in debate session logs

### HypothesisContext

**Purpose:** Comprehensive context passed to Hypothesis Generator for informed hypothesis generation.

**Location:** `aust/src/data_models/hypothesis.py`

**Key Attributes:**
- `iteration_number`: int - Current inner loop iteration
- `past_results`: list[dict] | None - Summaries of previous iteration results
- `evaluator_feedback`: str | None - Feedback from previous experiment evaluation
- `retrieved_papers`: list[dict] | None - Papers retrieved via RAG for this iteration
- `memory_entries`: list[dict] | None - Relevant entries from long-term memory (future)
- `seed_template`: dict | None - Seed hypothesis template (iteration 1 only)
- `task_spec`: dict - Full TaskSpec dictionary
- `critic_feedback_summaries`: list[dict] | None - Recent critic feedback for learning

**Usage:**
- Built by InnerLoopOrchestrator before hypothesis generation
- Passed to HypothesisRefinementWorkforce
- Updated during debate with new retrieved papers
- Enables evidence-based, context-aware hypothesis refinement

**Evolution Across Iterations:**
- **Iteration 1**: Only has seed_template + task_spec
- **Iteration 2+**: Adds past_results, evaluator_feedback, retrieved_papers
- **During debate**: retrieved_papers accumulates across debate rounds

**Persistence:** Not directly persisted; reconstructed for each iteration

### TaskSpec

**Purpose:** Structured specification of the unlearning task under test, extracted by TaskParserAgent.

**Location:** `aust/src/data_models/task_spec.py`

**Key Attributes:**
- `task_type`: str - "data_based_unlearning" or "concept_erasure"
- `model_name`: str - e.g., "stable-diffusion", "sdxl", "flux"
- `model_version`: str | None - e.g., "1.4", "turbo"
- `base_model_path`: str - Path to original model checkpoint
- `unlearned_model_path`: str - Path to unlearned model checkpoint
- `unlearned_target`: str | None - What was unlearned (e.g., "Van Gogh", "nudity")
- `unlearning_method`: str | None - Method used (e.g., "ESD", "concept-ablation")
- `user_prompt`: str | None - Original user input prompt
- `additional_params`: dict - Any extra task-specific parameters

**Validation:**
- task_type must be valid enum value
- Both model paths must be provided and non-empty
- model_name is required

**Usage:**
- Created by TaskParserAgent from user prompt
- Passed to InnerLoopOrchestrator
- Included in HypothesisContext for all agents
- Determines RAG collection selection (image vs. text tasks)

**Assembly:**
```python
TaskSpec.assemble(
    task_type="concept_erasure",
    base_model_path="/data/models/sd14.safetensors",
    unlearned_model_path="/data/models/sd14-esd-vangogh.safetensors",
    user_prompt="Attack Van Gogh erasure with ESD",
    parser_result={"model_name": "stable-diffusion", ...},
    overrides={"unlearning_method": "esd"}
)
```

**Persistence:**
- Saved in InnerLoopState
- Part of all iteration logs

### ExitCondition

**Purpose:** Enum representing why the inner loop terminated.

**Location:** `aust/src/data_models/loop_state.py`

**Values:**
- `VULNERABILITY_FOUND` - Successful vulnerability discovered (confidence ≥ threshold)
- `MAX_ITERATIONS` - Reached maximum iteration limit without finding vulnerability
- `USER_STOPPED` - User interrupted execution (Ctrl+C)
- `ERROR` - Unhandled error forced termination

**Usage:**
- Set by InnerLoopOrchestrator when loop exits
- Included in final state and attack trace
- Drives downstream reporting logic

---

### InnerLoopState

**Purpose:** Comprehensive state tracking for the entire inner loop execution. Enables checkpointing and resumption.

**Location:** `aust/src/data_models/loop_state.py`

**Key Attributes:**
- `task_id`: str - Unique task identifier
- `task_type`: str - Task type
- `task_description`: str - Human-readable task description
- `task_spec`: dict - Full TaskSpec as dictionary
- `max_iterations`: int - Maximum allowed iterations
- `current_iteration`: int - Current iteration number (0 = not started)
- `enable_debate`: bool - Whether multi-round debate is enabled
- `iterations`: list[IterationResult] - All completed iterations
- `exit_condition`: ExitCondition | None - Why loop exited
- `exit_message`: str - Detailed exit explanation
- `vulnerability_found`: bool - Was vulnerability discovered?
- `highest_vulnerability_confidence`: float - Best confidence score across all iterations
- `output_dir`: Path - Where outputs are stored
- `attack_trace_file`: Path | None - Path to generated attack trace
- `started_at`: datetime - Loop start time
- `completed_at`: datetime | None - Loop completion time

**Computed Properties:**
- `total_duration_seconds`: float - Total execution time
- `latest_iteration`: IterationResult | None - Most recent iteration

**Methods:**
- `should_continue() -> tuple[bool, str]` - Check if loop should continue
- `add_iteration_result(result: IterationResult)` - Add completed iteration
- `get_past_results_summary(num_recent: int) -> list[dict]` - Get recent iteration summaries
- `get_evaluator_feedback() -> str | None` - Get latest evaluator feedback
- `mark_complete(condition: ExitCondition, message: str)` - Mark loop as complete
- `save_to_file(path: Path)` - Persist state to JSON
- `load_from_file(path: Path) -> InnerLoopState` - Restore from JSON (classmethod)

**Persistence:**
- Auto-saved after each iteration to `aust/outputs/{task_id}/loop_state.json`
- Enables resume via `InnerLoopOrchestrator.resume_from_state()`

**Serialization:**
- Uses Pydantic's `model_dump(mode="json")` for JSON export
- All datetime fields converted to ISO-8601 strings
- Path fields converted to strings
- DebateSessions fully serialized with all exchanges

## Model Relationships Summary

```
InnerLoopState (1) ──> (N) IterationResult
    │                          │
    │                          ├──> (1) Hypothesis
    │                          ├──> (1) DebateSession
    │                          ├──> (N) RAG Queries (strings)
    │                          ├──> (N) Retrieved Papers (dicts)
    │                          ├──> (1) ExperimentResult (dict, placeholder)
    │                          └──> (1) Evaluator Feedback (string, placeholder)
    │
    └──> (1) TaskSpec (embedded as dict)

DebateSession (1) ──> (N) DebateExchange
    │                          │
    │                          ├──> (1) Initial Hypothesis
    │                          ├──> (1) CriticFeedback
    │                          ├──> (1) Refined Hypothesis (optional)
    │                          ├──> (N) RAG Queries (strings)
    │                          └──> (N) Retrieval Context (papers, dicts)
    │
    └──> (1) Final Hypothesis

CriticFeedback
    ├── novelty_score: float
    ├── feasibility_score: float
    ├── rigor_score: float
    ├── strengths: list[str]
    ├── weaknesses: list[str]
    ├── suggestions: list[str]
    └── overall_assessment: str

HypothesisContext
    ├── iteration_number: int
    ├── past_results: list[dict] | None
    ├── evaluator_feedback: str | None
    ├── retrieved_papers: list[dict] | None
    ├── seed_template: dict | None
    ├── task_spec: dict
    └── critic_feedback_summaries: list[dict] | None
```
