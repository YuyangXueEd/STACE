# Story 4.5: Judge Agent Implementation

## Status

✅ **COMPLETE** - Full judge workforce implementation with committee evaluation 

## Story

**As a** researcher,
**I want** Judge agents that evaluate generated reports from their persona's perspective,
**so that** the system provides multi-faceted evaluation of findings.

## Acceptance Criteria

1. Judge agent implemented in `agents/judge.py` that can instantiate any persona from `judge_personas.yaml`
2. Judge accepts inputs: generated report, attack traces, experiment results
3. Judge generates structured evaluation: summary assessment, strengths, weaknesses, scoring (per persona's dimensions), recommendations
4. Each judge evaluation saved to `aust/outputs/judgments/judge_{persona_name}_{run_id}.md`
5. Judges run independently (can be parallelized if time permits)
6. All judge outputs aggregated into `aust/outputs/judgments/summary_{run_id}.md`

### Completion Notes List

1. **Persona-Driven Architecture**: JudgeAgent dynamically loads personas from YAML config
   - Persona fields: id, name, role_description, tone, focus_areas, evaluation_criteria, scoring_dimensions
   - Per-persona model overrides supported (model_name, model_config)
   - Persona validation ensures required fields present

2. **LLM Runner with Model Caching**:
   - Default model: openai/gpt-5-nano via OpenRouter
   - Per-persona model resolution with fallback to default
   - Model instance caching to avoid recreating identical models
   - CAMEL ChatAgent wrapper for message handling
   - Implementation: lines 301-370

3. **Structured Evaluation Output**:
   - JSON response format with strict schema
   - Fields: summary, strengths, weaknesses, recommendations, scores, overall_rating
   - Response parsing handles code fences and JSON extraction
   - Graceful failure handling with fallback evaluations

4. **Committee Workflow**:
   - `run_committee()` executes all personas sequentially
   - CommitteeAggregate calculates average scores across dimensions
   - Individual and aggregate evaluations persisted
   - Integration in main.py outer loop (lines 550-628)

5. **Data Models**:
   - JudgeEvaluation: Single persona evaluation result
   - JudgeScore: Dimension-specific score with justification
   - CommitteeAggregate: Aggregated results from all personas
   - Implementation in `aust/src/data_models/judge.py`

6. **Integration Points**:
   - Called by `_run_judge_committee()` in main.py after report generation
   - Attack trace and experiment results passed as context
   - Judge feedback can be captured for outer loop refinement
   - Output directory: `outputs/judgments/`
