## Story

**As a** researcher,
**I want** a collaborative Hypothesis Refinement Workforce that uses iterative debate between a Hypothesis Generator and Critic agent with self-improving Chain-of-Thought reasoning,
**so that** the system autonomously generates high-quality, testable vulnerability hypotheses through structured adversarial refinement.

## Acceptance Criteria

### Core Workforce Implementation (AC1-AC3)
1. Hypothesis Refinement Workforce implemented as a cohesive module in `agents/hypothesis_workforce.py`
2. Workforce uses CAMEL-AI Workforce pattern to coordinate two specialized agents:
   - **Hypothesis Generator Agent**: Proposes vulnerability hypotheses, in the first round, build hypothesis based-on template
   - **Critic Agent**: Challenges and refines hypotheses through adversarial questioning
3. Workforce implements **configurable multi-round debate cycle** (default: 3 rounds) with RAG-enhanced self-improving CoT reasoning:
   - **Round N**: Generator creates/refines hypothesis → Critic evaluates → QueryGenerator generates targeted RAG queries based on feedback → RAG retrieves relevant papers → Evidence injected into next round
   - **Convergence**: Debate continues until quality threshold met (avg score ≥ 0.85) or max rounds reached
   - **Evidence Accumulation**: Retrieved papers accumulate across rounds for increasingly grounded refinements

### Hypothesis Generation & Data Models (AC4-AC6)

4. Hypothesis Generator accepts context inputs: task type, past experiment results, evaluator feedback, retrieved papers chunks from RAG
5. In the first round, the generator uses seed templates (3-5 known attack patterns) with LLM-based variation for initial hypothesis generation
6. Hypothesis data model implemented with structured format including: attack method, target, experiment parameters, expected outcome, confidence score, novelty score, reasoning trace

### Critic Agent Debate Mechanics (AC7-AC9)

7. Critic Agent challenges hypotheses on three dimensions:
   - **Novelty**: Is this just repeating seed templates or past attempts?
   - **Feasibility**: Can this hypothesis actually be executed with available tools/datasets?
   - **Rigor**: Is the expected outcome testable and well-defined?
8. Critic provides structured feedback format: strengths, weaknesses, specific suggestions for improvement, including citations to retrieved papers when applicable
9. Hypothesis Generator incorporates critic feedback using CoT reasoning to produce improved hypothesis with explicit reasoning trace

### Workflow Integration & Logging (AC10-AC12)

10. Workforce activates after first inner loop iteration (when feedback from evaluator is available)
11. Full debate exchange logged to `outputs/{task_id}/debates/iteration_{n}.json` including:
    - Initial hypothesis with reasoning trace
    - Critic feedback with evaluation scores and RAG query list + retrieved paper IDs
    - Refined hypothesis with incorporated improvements
    - Debate metadata (rounds, timestamps, model calls)
12. Final hypothesis from workforce integrates with existing Inner Loop Orchestrator state machine

### Key Implementation Details

#### Actual Implementation Architecture

**1. Multi-Round Debate with RAG Integration**
- Configurable debate rounds via `max_debate_iterations` (default: 3)
- Each round: Generator → Critic → QueryGenerator → RAG → Next round
- Evidence accumulates across rounds for increasingly grounded refinements
- Convergence detection via improvement delta (< 0.05 triggers early exit)

**2. QueryGeneratorAgent (Key Innovation)**
- Generates 1-3 targeted RAG queries per debate round
- Based on: current hypothesis, critic feedback, task context
- Intelligently selects RAG collection (any_to_v for image tasks, any_to_t for text tasks)
- Retrieved papers injected into next refinement round
- Query log saved to `outputs/{task_id}/queries/`

**3. DebateSession Persistence**
- Full debate history logged to `outputs/{task_id}/debates/debate_iteration_{N}.json`
- Includes: all hypotheses, critic feedback, RAG queries, retrieved papers, improvement deltas
- Each DebateExchange tracks: round number, generator/critic responses, retrieval context
- Enables post-hoc analysis of debate convergence patterns

**4. Workflow Logic**
- **Iteration 1**: Simple generation from seed template (no debate - fast & cheap)
- **Iteration 2+**: Full multi-round debate with RAG enhancement
- Debate continues until: quality threshold met (≥0.85) OR max rounds reached OR improvement stalls

#### Actual Workflow (Multi-Round Debate with RAG)

**Iteration 1 (Simple Generation - No Debate)**
```
1. Load seed template from configs/tasks/{task_type}.yaml
2. Generate single hypothesis using template + LLM variation
3. Return immediately (no critic, no RAG - fast & cheap)
4. Duration: ~5-10 seconds
```

**Iteration 2+ (Multi-Round RAG-Enhanced Debate)**
```
Round 1:
  1. Generator → Create initial hypothesis from context
  2. Critic → Evaluate (novelty=0.6, feasibility=0.7, rigor=0.8)
  3. QueryGenerator → Generate 3 targeted queries based on feedback
     Example: "gradient manipulation attacks", "concept erasure vulnerabilities"
  4. RAG → Retrieve top-5 papers per query (15 chunks total)
  5. Context updated with retrieved papers

Round 2:
  6. Generator → Refine hypothesis with evidence from papers
  7. Critic → Re-evaluate (novelty=0.8, feasibility=0.8, rigor=0.9)
  8. improvement_delta = 0.15 (significant improvement)
  9. QueryGenerator → Generate follow-up queries for remaining gaps
  10. RAG → Retrieve additional papers

Round 3:
  11. Generator → Further refinement with accumulated evidence
  12. Critic → Final evaluation (novelty=0.9, feasibility=0.9, rigor=0.95)
  13. avg_score = 0.92 ≥ threshold (0.85) → CONVERGED
  14. Return final hypothesis + full debate log

Duration: ~60-90 seconds for 3 rounds
```

**Convergence Conditions**:
- Quality threshold met: `average_score ≥ 0.85`
- Max rounds reached: `round_number ≥ max_debate_iterations`
- Improvement stalls: `abs(improvement_delta) < 0.05`

### Error Handling Strategy

**API Retry Logic (from Self-Improving CoT pattern):**
```python
import time
import functools
from typing import Callable

def retry_on_error(max_retries: int = 3, initial_delay: float = 1.0) -> Callable:
    def decorator(func: Callable) -> Callable:
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            delay = initial_delay
            for attempt in range(max_retries + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_retries:
                        logger.error(f"Max retries reached for {func.__name__}: {e}")
                        raise
                    logger.warning(f"Attempt {attempt + 1} failed: {e}. Retrying in {delay}s...")
                    time.sleep(delay)
                    delay *= 2  # Exponential backoff
            raise
        return wrapper
    return decorator
```

**Debate Failure Handling:**
- If critic rejects hypothesis entirely: fallback to seed template variation
- If generator cannot improve after 2 rounds: accept best hypothesis from debate history
- If API fails during debate: save partial debate log, retry from last successful round
- If debate diverges (quality decreasing): terminate early, use Round 1 refined hypothesis

### Performance Considerations

- Debate adds 4-6 LLM calls per iteration (2 rounds × 2-3 calls each)
- Implement rate limiting in OpenRouter client wrapper
