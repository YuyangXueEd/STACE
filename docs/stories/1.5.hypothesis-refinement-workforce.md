# Story 1.4-1.5: Hypothesis Refinement Workforce (Merged)

## Status

Draft

## Story

**As a** researcher,
**I want** a collaborative Hypothesis Refinement Workforce that uses iterative debate between a Hypothesis Generator and Critic agent with self-improving Chain-of-Thought reasoning,
**so that** the system autonomously generates high-quality, testable vulnerability hypotheses through structured adversarial refinement.

## Acceptance Criteria

### Core Workforce Implementation (AC1-AC3)

1. Hypothesis Refinement Workforce implemented as a cohesive module in `agents/hypothesis_workforce.py`
2. Workforce uses CAMEL-AI Workforce pattern to coordinate two specialized agents:
   - **Hypothesis Generator Agent**: Proposes vulnerability hypotheses
   - **Critic Agent**: Challenges and refines hypotheses through adversarial questioning
3. Workforce implements 2-round debate cycle with self-improving CoT reasoning:
   - **Round 1**: Generator creates initial hypothesis → Critic provides feedback → Generator refines
   - **Round 2**: Refined hypothesis → Critic validates/challenges → Final hypothesis output

### Hypothesis Generation & Data Models (AC4-AC6)

4. Hypothesis Generator accepts context inputs: task type, past experiment results, evaluator feedback, retrieved papers chunks from RAG
5. In the first round (outer loop), the generator uses seed templates (3-5 known attack patterns) with LLM-based variation for initial hypothesis generation
6. Hypothesis data model implemented with structured format including: attack method, target, experiment parameters, expected outcome, confidence score, novelty score, reasoning trace

### Critic Agent Debate Mechanics (AC7-AC9)

7. Critic Agent challenges hypotheses on three dimensions:
   - **Novelty**: Is this just repeating seed templates or past attempts?
   - **Feasibility**: Can this hypothesis actually be executed with available tools/datasets?
   - **Rigor**: Is the expected outcome testable and well-defined?
8. Critic provides structured feedback format: strengths, weaknesses, specific suggestions for improvement
9. Hypothesis Generator incorporates critic feedback using CoT reasoning to produce improved hypothesis with explicit reasoning trace

### Workflow Integration & Logging (AC10-AC12)

10. Workforce activates after first inner loop iteration (when feedback from evaluator is available)
11. Full debate exchange logged to `outputs/{task_id}/debates/iteration_{n}.json` including:
    - Initial hypothesis with reasoning trace
    - Critic feedback with evaluation scores
    - Refined hypothesis with incorporated improvements
    - Debate metadata (rounds, timestamps, model calls)
12. Final hypothesis from workforce integrates with existing Inner Loop Orchestrator state machine

- Configure evaluation criteria in system message

**3. Self-Improving CoT** (from `self_improving_cot_generation.md`):
- Implement iterative refinement loop: Generate → Evaluate → Refine
- Use `reason_agent` for hypothesis generation with CoT prompts
- Use `evaluate_agent` (Critic) for quality assessment
- Track improvement across rounds with score thresholds
- Implement retry logic with exponential backoff for API stability

**Debate Cycle Implementation:**
```python
# Round 1: Initial generation and critique
initial_hypothesis = generator.generate_hypothesis(context)
round1_feedback = critic.critique_hypothesis(initial_hypothesis, context)
refined_hypothesis = generator.refine_hypothesis(initial_hypothesis, round1_feedback)

# Round 2: Validate refinement
round2_feedback = critic.critique_hypothesis(refined_hypothesis, context)
final_hypothesis = generator.refine_hypothesis(refined_hypothesis, round2_feedback)

# Check quality threshold (average of novelty, feasibility, rigor scores)
quality_score = (round2_feedback.novelty_score +
                 round2_feedback.feasibility_score +
                 round2_feedback.rigor_score) / 3.0
```

from pydantic import BaseModel, Field, validator
from datetime import datetime, timezone
from typing import Optional

class Hypothesis(BaseModel):
    hypothesis_id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    attack_type: str  # "membership_inference", "model_inversion", etc.
    description: str
    rationale: str
    reasoning_trace: str  # NEW: CoT reasoning from generator
    target: str
    expected_outcome: str
    confidence_score: float = Field(ge=0.0, le=1.0)
    novelty_score: float = Field(ge=0.0, le=1.0)
    generated_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))

    @validator('generated_at', pre=True, always=True)
    def ensure_timezone_aware(cls, v):
        if v and v.tzinfo is None:

        return v

    weaknesses: list[str]
    suggestions: list[str]

### Error Handling Strategy

**API Retry Logic (from Self-Improving CoT pattern):**
```python
import time
import functools
from typing import Callable

def retry_on_error(max_retries: int = 3, initial_delay: float = 1.0) -> Callable:
    def decorator(func: Callable) -> Callable:
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            delay = initial_delay
            for attempt in range(max_retries + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_retries:
                        logger.error(f"Max retries reached for {func.__name__}: {e}")
                        raise
                    logger.warning(f"Attempt {attempt + 1} failed: {e}. Retrying in {delay}s...")
                    time.sleep(delay)
                    delay *= 2  # Exponential backoff
            raise
        return wrapper
    return decorator
```

**Debate Failure Handling:**
- If critic rejects hypothesis entirely: fallback to seed template variation
- If generator cannot improve after 2 rounds: accept best hypothesis from debate history
- If API fails during debate: save partial debate log, retry from last successful round
- If debate diverges (quality decreasing): terminate early, use Round 1 refined hypothesis

### Performance Considerations

**OpenRouter Rate Limits (NFR13):**
- Debate adds 4-6 LLM calls per iteration (2 rounds × 2-3 calls each)
- Implement rate limiting in OpenRouter client wrapper
- Add exponential backoff retry logic (3 attempts, 1s → 2s → 4s delays)
- Monitor token usage: ~3000-5000 tokens per debate cycle

**Debate Convergence Optimization:**
- Implement early termination if quality threshold met (e.g., avg score > 0.85)
- Detect minimal improvement (score delta < 0.05) and skip Round 2
- Cache critic evaluations for identical hypotheses (unlikely but possible)
