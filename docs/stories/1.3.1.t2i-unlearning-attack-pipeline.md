# Story 1.3.1: T2I Unlearning Attack Pipeline

## Status

Ready for Review

## Story

**As a** researcher,
**I want** to integrate ESD unlearning method as an MCP toolkit and create an agent to orchestrate concept erasure across multiple T2I models,
**so that** I can programmatically erase concepts from various text-to-image models (Stable Diffusion, SDXL, Flux) via natural language prompts, with Stable Diffusion as the default.

## Acceptance Criteria

1. ESD repository cloned to `external/esd/` (from official GitHub repository)
2. ConceptUnlearnToolkit in `aust/src/toolkits/concept_unlearn_toolkit.py` wraps ESD as MCP toolkit with FunctionTools supporting multiple T2I models. Read the README file carefully in the repo.
3. Support for T2I models: Stable Diffusion (default), Stable Diffusion XL (SDXL), and Flux
4. ConceptUnlearnAgent in `aust/src/agents/concept_unlearn_agent.py` receives prompts like "using {unlearning_method} to erase {concept} [from {model}] [with {method_variant}]" and triggers appropriate tool calls. When executing, please print the detailed log from the original function print.
5. Support for ESD method with optional method variants (e.g., 'xattn', 'noxattn', 'full')
6. Default parameters when model/variant not specified in prompt (model: Stable Diffusion, variant: 'xattn' for ESD)
7. Model-specific configuration and checkpoint management (`data/unlearned_models/esd/{model}/{concept}/`)
8. Toolkit designed to be extensible for future unlearning methods (SalUn, Receler, etc.) and T2I models

**Tool Function Signature Example:**
```python
def erase_concept_esd(
    concept: str,
    model: str = "stable-diffusion",
    method_variant: str = "xattn",
    guidance_scale: float = 7.5,
    learning_rate: float = 1e-5,
    num_steps: int = 1000,
    output_dir: str = "data/unlearned_models/esd"
) -> str:
    """
    Erase a concept from a T2I model using ESD method.

    Args:
        concept: Concept to erase (e.g., 'Van Gogh', 'nudity')
        model: T2I model to use ('stable-diffusion', 'sdxl', 'flux'). Default: 'stable-diffusion'
        method_variant: ESD method variant ('xattn', 'noxattn', 'full'). Default: 'xattn'
        guidance_scale: Guidance scale for diffusion (default: 7.5)
        learning_rate: Learning rate for unlearning (default: 1e-5)
        num_steps: Number of unlearning steps (default: 1000)
        output_dir: Output directory for unlearned checkpoint (model-specific subdirs created automatically)

    Returns:
        Path to unlearned model checkpoint
        
    Raises:
        ValueError: If model is not supported or concept is invalid
        RuntimeError: If CUDA OOM or subprocess execution fails
    """
```

```python
from pydantic import BaseModel, Field

class ModelConfig(BaseModel):
    """Configuration for supported T2I models."""
    name: str = Field(..., description="Model identifier (stable-diffusion, sdxl, flux)")
    model_id: str = Field(..., description="Hugging Face model ID")
    default_steps: int = Field(default=1000, description="Default num_steps for unlearning")
    default_guidance_scale: float = Field(default=7.5, description="Default guidance scale")
    
    class Config:
        frozen = True

# Supported models
MODELS = {
    "stable-diffusion": ModelConfig(
        name="stable-diffusion",
        model_id="CompVis/stable-diffusion-v1-4",
        default_steps=1000,
        default_guidance_scale=7.5
    ),
    "sdxl": ModelConfig(
        name="sdxl",
        model_id="stabilityai/stable-diffusion-xl-base-1.0",
        default_steps=1000,
        default_guidance_scale=7.5
    ),
    "flux": ModelConfig(
        name="flux",
        model_id="black-forest-labs/FLUX.1-dev",
        default_steps=1000,
        default_guidance_scale=7.5
    )
}
```
