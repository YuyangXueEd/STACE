# Story 1.3.1: T2I Unlearning Attack Pipeline

## Status

Draft

## Story

**As a** researcher,
**I want** to integrate ESD unlearning method as an MCP toolkit and create an agent to orchestrate concept erasure,
**so that** I can programmatically erase concepts from Stable Diffusion models via natural language prompts.

## Acceptance Criteria

1. ESD repository cloned to `external/esd/` (from official GitHub repository)
2. ConceptUnlearnToolkit in `aust/src/toolkits/concept_unlearn_toolkit.py` wraps ESD as MCP toolkit with FunctionTools
3. ConceptUnlearnAgent in `aust/src/agents/concept_unlearn_agent.py` receives prompts like "using {unlearning_method} to erase {concept} [with {method_variant}]" and triggers appropriate tool calls
4. Support for ESD method with optional method variants (e.g., 'xattn', 'noxattn', 'full')
5. Default parameters when method variant not specified in prompt (e.g., default to 'xattn' for ESD)
6. Toolkit designed to be extensible for future unlearning methods (SalUn, Receler, etc.)

**Tool Function Signature Example:**
```python
def erase_concept_esd(
    concept: str,
    method_variant: str = "xattn",
    guidance_scale: float = 7.5,
    learning_rate: float = 1e-5,
    num_steps: int = 1000,
    output_dir: str = "data/unlearned_models/esd"
) -> str:
    """
    Erase a concept from Stable Diffusion using ESD method.

    Args:
        concept: Concept to erase (e.g., 'Van Gogh', 'nudity')
        method_variant: ESD method variant ('xattn', 'noxattn', 'full')
        guidance_scale: Guidance scale for diffusion (default: 7.5)
        learning_rate: Learning rate for unlearning (default: 1e-5)
        num_steps: Number of unlearning steps (default: 1000)
        output_dir: Output directory for unlearned checkpoint

    Returns:
        Path to unlearned model checkpoint
    """
```
