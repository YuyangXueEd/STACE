# Story 1.2: CAMEL-AI Integration in Dev Mode

## Status

Draft

## Story

**As a** developer,
**I want** to integrate CAMEL-AI in dev/editable mode and verify basic agent functionality,
**so that** we can create and modify agents as needed throughout development.

## Acceptance Criteria

1. CAMEL-AI is cloned/installed in `external/camel/` with `pip install -e external/camel`
2. Basic test agent (simple echo agent) successfully instantiates using CAMEL-AI's agent API
3. OpenRouter API integration is tested with at least one model call (GPT-4o or Claude 3.5)
4. Agent prompt configuration system is implemented in `configs/` directory (YAML or JSON format)
5. Basic agent interaction logging captures all LLM API calls and responses

## Tasks / Subtasks

- [ ] Clone and install CAMEL-AI in dev mode (AC: 1)
  - [ ] Clone CAMEL-AI repository to `external/camel/`
  - [ ] Install in editable mode: `pip install -e external/camel`
  - [ ] Verify installation by importing camel in Python
  - [ ] Document CAMEL-AI version/commit in requirements.txt as comment

- [ ] Create base agent wrapper (AC: 2)
  - [ ] Implement `agents/base_agent.py` wrapping CAMEL-AI's BaseAgent
  - [ ] Create simple echo agent for testing
  - [ ] Test agent instantiation and basic message passing
  - [ ] Add error handling for agent initialization failures

- [ ] Set up OpenRouter API integration (AC: 3)
  - [ ] Create OpenRouter client wrapper class
  - [ ] Implement API key loading from environment variable
  - [ ] Add retry logic with exponential backoff (2s, 4s, 8s)
  - [ ] Test API call with GPT-4o or Claude 3.5 Sonnet
  - [ ] Add timeout handling (60s per request)
  - [ ] Log all API calls (redact API key)

- [ ] Implement agent prompt configuration system (AC: 4)
  - [ ] Create config schema in `configs/prompts/` directory
  - [ ] Define YAML structure: agent_name, task_type, system_prompt, model, temperature, max_tokens
  - [ ] Implement configuration loader using PyYAML
  - [ ] Create sample prompt config file for testing
  - [ ] Add validation for required fields

- [ ] Set up agent interaction logging (AC: 5)
  - [ ] Extend logging framework to capture LLM interactions
  - [ ] Log: timestamp, agent_name, model, prompt, response, tokens, latency
  - [ ] Use JSON structured logging for machine parsing
  - [ ] Store logs in `outputs/agent_logs/` directory
  - [ ] Test logging with echo agent

## Dev Notes

### Relevant Source Tree

**Files to Create:**
- `agents/__init__.py`
- `agents/base_agent.py` - Base agent class wrapping CAMEL-AI
- `configs/prompts/` - Directory for agent prompt configs
- `configs/prompts/test_agent.yaml` - Sample config for testing

**External Dependencies:**
- `external/camel/` - CAMEL-AI in dev/editable mode

### Tech Stack

From [architecture/tech-stack.md](../architecture/tech-stack.md):

| Technology | Version | Purpose |
|-----------|---------|---------|
| CAMEL-AI | dev/editable (latest main) | Multi-agent orchestration |
| OpenRouter | N/A (API service) | LLM/VLM access |
| PyYAML | 6.0.1 | YAML config parsing |

**OpenRouter API:**
- Base URL: https://openrouter.ai/api/v1
- Authentication: Bearer token (`Authorization: Bearer $OPENROUTER_API_KEY`)
- Timeout: 60s per request
- Retry: 3 attempts with exponential backoff

### Coding Standards

**Critical Rules:**
- Agent LLM calls must go through OpenRouter client wrapper (never call API directly)
- Never hardcode API keys - load from environment variables
- All logging must use Python logging module, not print()
- Type hints mandatory for all function signatures

### Component Specifications

From [architecture/components.md](../architecture/components.md):

**Base Agent Pattern:**
- All agents extend base_agent.py which wraps CAMEL-AI's BaseAgent
- Agents load config from `configs/prompts/{agent_name}_{task_type}.yaml`
- All LLM calls route through OpenRouter client wrapper
- Model selection configurable per agent via AgentPromptConfig

**Configuration Manager:**
- `load_agent_config(agent_name: str, task_type: str | None) -> AgentPromptConfig`
- YAML-based configs for human readability
- Validates configs against schemas on load
- Caches loaded configs to avoid repeated file reads

### Testing

- **Framework:** pytest 7.4.3
- **Location:** `tests/unit/test_base_agent.py`
- **Mock:** pytest-mock for OpenRouter API calls

**Test Cases:**
- Test CAMEL-AI imports successfully
- Test base agent instantiation
- Test echo agent message handling
- Test OpenRouter API call with mocked response
- Test config loading from YAML
- Test agent interaction logging
