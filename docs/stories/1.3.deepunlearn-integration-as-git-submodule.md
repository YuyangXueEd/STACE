# Story 1.3: DeepUnlearn Integration as Git Submodule

## Status

Completed

## Story

**As a** developer,
**I want** to integrate DeepUnlearn as a git submodule and wrap it as CAMEL-AI BaseToolkit with MCP FunctionTools,
**so that** agents can programmatically trigger unlearning, evaluation experiments, and orchestrate the complete DeepUnlearn pipeline through natural language prompts.

## Acceptance Criteria

1. DeepUnlearn repository is added as git submodule in `external/DeepUnlearn/`
2. DeepUnlearn dependencies are integrated into requirements.txt without conflicts
3. BaseToolkit wrapper in `aust/src/toolkits/deepunlearn_toolkit.py` (DeepUnlearnToolkit) exposes pipeline operations: `prepare_data_splits`, `generate_initial_models`, `link_model_initializations`, `train_model`, `run_unlearning_method`, and `materialize_artifacts`
4. BaseToolkit wrapper in `aust/src/toolkits/deepunlearn_evaluation_toolkit.py` (DeepUnlearnEvaluationToolkit) exposes evaluation functions: `evaluate_model_checkpoint`, `compute_indiscernibility`, `compute_accuracy_retention`, `compute_sape_score`, and `compute_model_weight_distance`
5. DeepUnlearnOrchestrator agent in `aust/src/agents/deepunlearn_orchestrator.py` integrates pipeline toolkit and provides natural language interface to execute pipeline steps
6. Pipeline script `aust/scripts/run_deepunlearn_pipeline.py` provides command-line interface to orchestrate complete DeepUnlearn workflow from data preparation to model materialization
7. DeepUnlearnEvaluationAgent in `aust/src/agents/deepunlearn_evaluator.py` integrates evaluation toolkit and provides natural language interface to evaluate trained/unlearned model checkpoints
8. Evaluation script `aust/scripts/run_deepunlearn_eval.py` provides command-line interface to evaluate model checkpoints and compute comparison metrics
9. Error handling for pipeline failures, evaluation errors, GPU unavailability, invalid parameters, and DeepUnlearn subprocess errors

**DeepUnlearnToolkit Pipeline Functions:**
- `prepare_data_splits(dataset)` - Generate train/retain/forget/val splits for dataset(s)
- `generate_initial_models(model, num_classes, model_seeds, img_size)` - Create untrained model initializations
- `link_model_initializations()` - Symlink models to dataset folders
- `train_model(dataset, model, method, model_seed, random_state, overrides, save_path, timeout)` - Train baseline model
- `run_unlearning_method(method, dataset, model, model_seed, random_state, overrides, save_path, timeout)` - Apply unlearning method
- `materialize_artifacts(dataset, model, train_method, unlearn_method, model_seed, data_dir)` - Copy checkpoints and splits to project data directory

**DeepUnlearnEvaluationToolkit Evaluation Functions:**
- `evaluate_model_checkpoint(model_path, model_type, dataset, batch_size, random_state, device, split_index, forget_index)` - Evaluate checkpoint on all splits with retain/forget/val/test accuracy and MIA scores
- `compute_indiscernibility(mia_score)` - Calculate indiscernibility metric (1 - |MIA-0.5| / 0.5) from MIA score
- `compute_accuracy_retention(new_accuracy, reference_accuracy)` - Calculate retention ratio between unlearned and reference accuracy
- `compute_run_time_efficiency(new_time_seconds, reference_time_seconds)` - Calculate run-time efficiency relative to reference experiment
- `compute_sape_score(a, b)` - Symmetric absolute percentage error between two values
- `compute_model_weight_distance(model_a_path, model_b_path, model_type, dataset, device)` - L2 and normalized L2 distance between checkpoints

**DeepUnlearnEvaluationAgent Key Features:**
- Default prompt evaluates all metrics for both original and unlearned checkpoints
- EvaluationInputs dataclass bundles dataset, checkpoint_model_type, splits_dir, original_model, unlearned_model, batch_size, random_state, device, and optional LIRA indices
- Context block builder attaches artifact paths to prompts
- Custom prompt support via --prompt flag with automatic context appending
- Multi-turn conversation with tool call tracking
- JSON transcript persistence to data/deepunlearn_evaluation_\<timestamp\>.json

**Evaluation Script (run_deepunlearn_eval.py) Features:**
- Required flags: --splits-dir, --original-model, --unlearned-model
- Optional metric targeting via repeatable --metric flag
- LIRA support with --split-index and --forget-index (must be provided together)
- Path resolution and validation (FileNotFoundError if missing)
- Auto-generates structured prompt with context block
- Pass-through args for model backend (platform, type, URL, API key, temperature)
- Toolkit configuration (deepunlearn-dir, project-root, timeout, verbose)
