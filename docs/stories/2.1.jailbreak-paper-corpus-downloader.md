# Story 2.1: Jailbreak Paper Corpus Downloader & Storage

## Status

Completed

## Story

**As a** researcher,
**I want** to automatically download and categorize jailbreak attack papers from Awesome-Multimodal-Jailbreak repository,
**so that** the RAG system has a comprehensive knowledge base of attack methods for hypothesis generation.

## Acceptance Criteria

1. Paper downloader script in `aust/utils/download_jailbreak_papers.py` scrapes paper links from Awesome-Multimodal-Jailbreak repository (https://github.com/liuxuannan/Awesome-Multimodal-Jailbreak/blob/main/README.md). All the attacking papers are in `## 😈JailBreak Attack` section and in arxiv url. 
2. Papers categorized according to survey taxonomy: Input-level attacks, Encoder-level attacks, Generator-level attacks, Output-level attacks (matching Table III categories). According to `Any-to-T` or `Any-to-V` create folders.
3. Paper corpus directory structure: `.papers/{Any-to-X}/input_level/`, `.papers/{Any-to-X}/encoder_level/`, `.papers/{Any-to-X}/generator_level/`, `.papers/{Any-to-X}/output_level/`
4. Paper metadata file `.papers/paper_metadata.json` contains: title, authors, venue, year, arxiv_id, pdf_url, category, subcategory, file_path for each paper
5. Rate limiting and retry logic for arxiv.org downloads (max 3 requests/second, exponential backoff)
6. Download progress tracking: skip already downloaded papers, resume on failure

**Model Type to Category Mapping:**
- **Any-to-T** (Text output): IT2T, T2T, V2T, AnyModal2T
- **Any-to-V** (Visual output): T2I, IT2I, V2I, AnyModal2I

**Arxiv Download Strategy:**
1. Parse markdown table in `## 😈JailBreak Attack` section
2. Extract arxiv URLs from table cells (all attacking papers are in arxiv URL format)
3. Extract arxiv ID from URL: https://arxiv.org/abs/2024.xxxxx → 2024.xxxxx
4. Download PDF from https://arxiv.org/pdf/{arxiv_id}.pdf
5. Classify paper by model type (from table) → Any-to-T or Any-to-V
6. Save to appropriate directory: `.papers/{model_type_category}/{attack_level}/{subcategory}/{arxiv_id}.pdf`

**Error Handling:**
- HTTP 404: Log warning, mark as failed, continue
- HTTP 429: Rate limit hit, sleep for retry-after header duration
- HTTP 500: Server error, exponential backoff
- Corrupted PDF: Log error, mark as failed, save error message
- Network timeout: Retry with exponential backoff
