# Story 2.1: Jailbreak Paper Corpus Downloader & Storage

## Status

Draft

## Story

**As a** researcher,
**I want** to automatically download and categorize jailbreak attack papers from Awesome-Multimodal-Jailbreak repository,
**so that** the RAG system has a comprehensive knowledge base of attack methods for hypothesis generation.

## Acceptance Criteria

1. Paper downloader script in `aust/scripts/download_jailbreak_papers.py` scrapes paper links from Awesome-Multimodal-Jailbreak repository (https://github.com/liuxuannan/Awesome-Multimodal-Jailbreak)
2. Papers categorized according to survey taxonomy: Input-level attacks, Encoder-level attacks, Generator-level attacks, Output-level attacks (matching Table III categories)
3. Paper corpus directory structure: `rag/papers/input_level/`, `rag/papers/encoder_level/`, `rag/papers/generator_level/`, `rag/papers/output_level/`
4. Paper metadata file `rag/paper_metadata.json` contains: title, authors, venue, year, arxiv_id, pdf_url, category, subcategory, file_path for each paper
5. PDF parsing utility in `aust/src/utils/pdf_parser.py` extracts text from PDFs (using PyMuPDF or pdfplumber)
6. Extracted paper text stored in `rag/papers_text/` with structured sections (Abstract, Introduction, Method, Experiments, Conclusion)
7. Rate limiting and retry logic for arxiv.org downloads (max 3 requests/second, exponential backoff)
8. Download progress tracking: skip already downloaded papers, resume on failure

  - [ ] Papers saved as: `{category}/{subcategory}/{arxiv_id}.pdf`
  - [ ] Example: `rag/papers/input_level/prompt_engineering/2024.12345.pdf`
- [ ] Paper metadata file created (AC: 4)
  - [ ] PaperMetadata dataclass: title, authors, venue, year, arxiv_id, pdf_url, category, subcategory, file_path, download_date, abstract

  - [ ] PDFParser class in `aust/src/utils/pdf_parser.py`
  - [ ] `parse_pdf(pdf_path)` returns structured text with sections
  - [ ] Section detection: Abstract, Introduction, Method, Experiments, Conclusion (heuristic-based)
  - [ ] Text cleaning: remove headers, footers, page numbers, references formatting
  - [ ] PyMuPDF (fitz) as primary backend, pdfplumber as fallback
  - [ ] Error handling: corrupted PDFs, encoding issues, empty pages

  - [ ] Text storage directory: `rag/papers_text/`

  - [ ] Exponential backoff on HTTP errors: 1s, 2s, 4s, 8s (max 5 retries)
  - [ ] HTTP status code handling: 429 (rate limit), 500 (server error), 404 (not found)
  - [ ] User-Agent header: "CAUST-PaperDownloader/1.0 (Research; mailto:user@example.com)"
  - [ ] Timeout: 30s per request
- [ ] Download progress tracking (AC: 8)
  - [ ] Progress file: `rag/.download_progress.json` tracks downloaded papers
  - [ ] DownloadProgress dataclass: arxiv_id, status (pending/downloading/completed/failed), last_attempt, error_message
  - [ ] Skip already downloaded papers (check PDF file existence + progress status)
  - [ ] Resume on failure: retry failed papers on next run
  - [ ] CLI flag: --force-redownload (ignore progress, redownload all papers)
  - [ ] Progress bar: tqdm integration for download status

**Arxiv Download Strategy:**
1. Extract arxiv ID from citation (e.g., `[arXiv'24]` in venue column)
2. If arxiv ID not found, search arxiv API by title + authors
3. Download PDF from https://arxiv.org/pdf/{arxiv_id}.pdf
4. Fallback: Use paper link from repository if provided

**Error Handling:**
- HTTP 404: Log warning, mark as failed, continue
- HTTP 429: Rate limit hit, sleep for retry-after header duration
- HTTP 500: Server error, exponential backoff
- Corrupted PDF: Log error, mark as failed, save error message
- Network timeout: Retry with exponential backoff
