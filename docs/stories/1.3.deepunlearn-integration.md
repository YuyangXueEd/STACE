# Story 1.3: DeepUnlearn Integration as Git Submodule

## Status

Draft

## Story

**As a** developer,
**I want** to integrate DeepUnlearn as a git submodule and wrap it as a CAMEL-AI MCP FunctionTool,
**so that** agents can programmatically trigger unlearning and evaluation experiments.

## Acceptance Criteria

1. DeepUnlearn repository is added as git submodule in `submodules/DeepUnlearn/`
2. DeepUnlearn dependencies are integrated into requirements.txt without conflicts
3. MCP FunctionTool wrapper in `tools/deepunlearn_tool.py` exposes at minimum: `unlearn_model(method, dataset, params)` and `evaluate_model(model, metrics)`
4. Successful test execution: trigger unlearning on a simple dataset and verify evaluation metrics are returned
5. Error handling for GPU unavailability, invalid parameters, and DeepUnlearn failures

## Tasks / Subtasks

- [ ] Add DeepUnlearn as git submodule (AC: 1)
  - [ ] Run `git submodule add <DeepUnlearn-repo-URL> submodules/DeepUnlearn`
  - [ ] Pin submodule to specific commit SHA
  - [ ] Update .gitmodules file
  - [ ] Document submodule commit hash in requirements.txt as comment
  - [ ] Test submodule clone: `git submodule update --init --recursive`

- [ ] Integrate DeepUnlearn dependencies (AC: 2)
  - [ ] Review DeepUnlearn's requirements.txt
  - [ ] Merge dependencies into main requirements.txt
  - [ ] Resolve version conflicts (especially PyTorch)
  - [ ] Test: `pip install -r requirements.txt` succeeds
  - [ ] Verify no dependency conflicts with CAMEL-AI

- [ ] Create MCP FunctionTool wrapper (AC: 3)
  - [ ] Create `tools/__init__.py`
  - [ ] Implement `tools/deepunlearn_tool.py` as MCP FunctionTool adapter
  - [ ] Implement `unlearn_model(method: str, dataset: str, params: dict) -> dict`
  - [ ] Implement `evaluate_model(model_path: str, metrics: list[str]) -> dict`
  - [ ] Translate hypothesis parameters to DeepUnlearn API calls
  - [ ] Return standardized ExperimentResult format

- [ ] Implement error handling (AC: 5)
  - [ ] Catch GPU unavailability errors (CUDA not available)
  - [ ] Validate input parameters (method exists, dataset path valid)
  - [ ] Handle DeepUnlearn exceptions and translate to custom exceptions
  - [ ] Add timeout handling (30 minutes per NFR6)
  - [ ] Log all errors with full context

- [ ] Create integration test (AC: 4)
  - [ ] Set up test with mock/small dataset
  - [ ] Trigger unlearning with simple method (e.g., gradient ascent)
  - [ ] Verify metrics returned: forget accuracy, retain accuracy
  - [ ] Test error cases: invalid method, missing dataset
  - [ ] Mock GPU execution if not available in test environment

### Component Specifications

From [architecture/components.md](../architecture/components.md):

**DeepUnlearn FunctionTool:**
- **Responsibility:** Adapter wrapping DeepUnlearn git submodule for data-based unlearning experiments
- **Key Interfaces:**
  - `unlearn_model(method: str, dataset: str, params: dict) -> dict` - Triggers unlearning
  - `evaluate_model(model_path: str, metrics: list[str]) -> dict` - Evaluates unlearned model
- **Dependencies:** DeepUnlearn git submodule, PyTorch 2.8.0, Kubernetes Job API (for GPU execution)
- **Implementation Notes:** Implements Adapter pattern to expose DeepUnlearn methods as standardized FunctionTool interface

**ExperimentResult Data Model:**
From [architecture/data-models.md](../architecture/data-models.md):
```python
class ExperimentResult(BaseModel):
    experiment_id: str
    tool_used: str  # "deepunlearn"
    method: str  # e.g., "SISA", "gradient_ascent"
    execution_status: str  # "success", "failed", "timeout"
    metrics: dict  # {"forget_accuracy": 0.23, "retain_accuracy": 0.89}
    outputs: dict  # Additional outputs (model checkpoints, logs)
    error_message: str | None
    execution_time_seconds: float
    gpu_job_id: str | None
```

### Coding Standards

### Error Handling

From [architecture/error-handling-strategy.md](../architecture/error-handling-strategy.md):

**Custom Exceptions:**
- `ExperimentExecutionException` - Raised when GPU job fails or times out
- Handle GPU unavailability with retry logic
- Timeout: 30 minutes per job (NFR6)

**Retry Logic:**
- Transient GPU unavailability: 3 retries with 5-minute backoff
- No retry for: invalid parameters, missing datasets
