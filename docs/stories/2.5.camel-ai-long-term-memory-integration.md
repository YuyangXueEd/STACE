# Story 2.5: CAMEL-AI Long-Term Memory Integration

## Status

✅ **COMPLETE** - Fully implemented with Attack Memory Cards pattern

## Story

**As a** researcher,
**I want** to integrate CAMEL-AI's long-term memory to store successful vulnerability discoveries,
**so that** the system learns from past successes across multiple runs.

## Acceptance Criteria

1. Memory system implemented in `memory/long_term_memory.py` using CAMEL-AI's memory API
2. Successful experiments (VULNERABILITY_FOUND) are stored with: hypothesis, experiment parameters, results, attack trace reference
3. Memory retrieval interface: `get_successful_attacks(task_type)` returns past successful attacks for given task
4. Hypothesis Generator queries memory at start of each run to inform initial hypothesis generation
5. Memory persisted to `aust/outputs/memory_store/` (survives container restarts per NFR14)

### Completion Notes List

1. **Design Decision**: Chose file-based "Attack Memory Cards" over CAMEL-AI VectorDB memory
   - Rationale: Simpler, human-readable, sufficient for MVP scale (<50 attacks)
   - Follows proven paper card pattern already in codebase
   - Easy upgrade path to semantic search if needed

2. **Memory Card Format**: Structured markdown with sections:
   - `[METADATA]`: Task info, confidence, timestamps
   - `[HYPOTHESIS]`: Attack type, summary, reasoning
   - `[EXPERIMENT_PARAMETERS]`: Prompts, seeds, hyperparameters
   - `[RESULTS]`: Detection rate, successful prompts
   - `[LESSONS_LEARNED]`: Key findings extracted from evaluator feedback

3. **Integration Points**:
   - Storage: `InnerLoopOrchestrator.run()` after VULNERABILITY_FOUND
   - Retrieval: `InnerLoopOrchestrator._build_hypothesis_context()` in iteration 1
   - Rendering: `HypothesisRefinementWorkforce._render_memory_entries()`

4. **Test Coverage**: 9 comprehensive unit tests (100% passing)
   - Initialization, creation, storage, retrieval
   - Filtering by task_type, confidence, max_results
   - Memory summary generation
   - Markdown serialization/deserialization
