# Story 1.0: Task Parser & Input Validation

## Status

New

## Story

As a developer,
I want to validate and parse user inputs into a structured TaskSpec,
so that the system only runs when at least the model details and the unlearned concept/content are specified.

## Acceptance Criteria

1. Task Input Parser implemented in `src/agents/task_parser_agent.py` with a Pydantic `TaskSpec` model
2. Accepts a natural language prompt and extracts required fields:
   - `model_name` (e.g., "Stable Diffusion", "SDXL", "Flux") and optional `model_version`
   - `unlearning_target` (concept/content to erase)
   - Optional: `unlearning_method` (e.g., "ESD"), `method_variant`, other hints
3. Validates required fields and returns actionable errors on failure with suggestions to match the example pattern
4. Parses example pattern: "A Stable Diffusion 1.4 model unlearned with concept Cat {with ESD method}" where braces denote optional info
5. Emits normalized `TaskSpec` JSON into `outputs/{task_id}/task_spec.json` with timezone-aware timestamps
6. Integrated pre-check in Inner Loop Orchestrator; loop does not start if validation fails; error is logged with guidance

## Tasks / Subtasks

- [ ] Implement `TaskSpec` (Pydantic) with fields and validators
- [ ] Implement `parse_task_input(text: str) -> TaskSpec` with deterministic regex first, then controlled LLM fallback
- [ ] Add CLI helper `scripts/parse_task.py` to create `TaskSpec` from a prompt and save to outputs
- [ ] Wire Inner Loop Orchestrator to require a valid `TaskSpec` before starting
- [ ] Unit tests: valid prompt, missing model, missing concept, ambiguous input (ensure helpful errors)

## Dev Notes

- Prefer deterministic parsing with explicit regex groups; keep one LLM fallback with a strict schema and temperature <= 0.2
- Normalize model names and method aliases (e.g., SD → Stable Diffusion) via small lookup table
- Contract:
  - Input: free-form text prompt
  - Output: `TaskSpec` object and JSON path
  - Errors: `TaskParseError` with code and remediation tips
- Logs: Save raw input, parsed fields, and validation status in `outputs/{task_id}/task_spec.log`
