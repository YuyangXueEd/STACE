# Story 1.6a: Attack Code Synthesis & Self-Repair Loop

## Status

Completed

## Story

As a researcher,
I want an agent that translates refined hypotheses into executable concept-erasure attack code and automatically repairs code on failures,
so that failed executions are retried up to 3–5 times before giving up, forming the complete step 3 of the inner loop.

## Acceptance Criteria

1. Code Synthesis agent implemented in `agents/code_synthesizer.py` using Ollama's `qwen3-coder:480b` model (256k context) with prompts specifically tailored for concept-erasure attacks
2. Execution runner uses CAMEL's `SubprocessInterpreter` (see `external/camel/docs/reference/camel.interpreters.subprocess_interpreter`) for sandboxed execution with timeouts (default 15 min, configurable); logs captured to `aust/outputs/{task_id}/runs/{run_id}/`
3. Code-repair-execute loop forms the complete step 3 of the inner loop: code synthesizer takes final hypothesis as input → generates code → executes → repairs on failure (retry budget: default 3, max 5)
4. The same agent handles both code synthesis and repair (no separate repair agent) - leverages 256k context window to maintain full synthesis → execute → repair history
5. Success/failure with artifacts and logs reported back to the Inner Loop Orchestrator
6. Agent references `external/diffusers` library for diffusion model implementation patterns

## Dev Notes

- **Inner Loop Integration**: This forms step 3 of the inner loop. Input is the final hypothesis from step 2 (hypothesis refinement). Output is execution results passed to step 4 (evaluation).
- **Code Synthesizer Agent**: Single agent handles both synthesis and repair (no separate repair agent needed). Agent uses developer-style prompting similar to `.bmad-core/agents/dev.md`. The 256k context window allows maintaining full synthesis → execute → repair history in one conversation, improving repair quality through context continuity.
- **Model**: Use Ollama's `qwen3-coder:480b` (480B parameters, 256k context window, deployed locally). This large model provides advanced reasoning for complex code generation and repair tasks.
- **Execution**: Use CAMEL's `SubprocessInterpreter` from `camel.interpreters.subprocess_interpreter`. Configure with timeout (default 15 min, configurable) and capture stdout/stderr.
- **Repair Loop**: Keep retries targeted - extract error snippet, propose minimal fix, re-run. Avoid oscillations by tracking applied fixes. Max retries: configurable (default 3, max 5).
- **Diffusers Reference**: Agent should reference `external/diffusers` library for diffusion model implementation patterns when generating concept-erasure attack code.
- **Security**: Enforce resource and security constraints (no network calls unless explicitly allowed; respect CUDA_VISIBLE_DEVICES).
- **Orchestrator Integration**: Provide clear return codes and structured errors to the orchestrator for next-step decisions.
