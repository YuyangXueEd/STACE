# Story 1.6a: Attack Code Synthesis & Self-Repair Loop

## Status

New

## Story

As a researcher,
I want an agent that translates hypotheses into executable attack code/scripts and automatically repairs code on failures,
so that failed executions are retried up to 3–5 times before giving up.

## Acceptance Criteria

1. Code Synthesis agent implemented in `agents/code_synthesizer.py` with prompts tailored for our toolkits and environment
2. Execution runner integrates a sandboxed subprocess with timeouts; logs captured to `outputs/{task_id}/runs/{run_id}/`
3. On failure (non-zero exit, exception), the agent inspects error logs and regenerates code with targeted fixes; retry budget configurable (default 3, max 5)
4. Success/failure with artifacts and logs reported back to the Inner Loop Orchestrator
5. Works for both concept-erasure attack scripts (ESD flows) and data-based DeepUnlearn wrappers

## Tasks / Subtasks

- [ ] Create `agents/code_synthesizer.py` with:
  - [ ] `synthesize(hypothesis, context) -> CodeArtifact`
  - [ ] `execute(artifact) -> RunResult` (sandboxed, with timeout)
  - [ ] `repair(artifact, error_log) -> CodeArtifact` (focused diff, not full rewrite)
- [ ] Implement prompt templates in `configs/prompts/code_synthesizer.yaml` with environment assumptions (paths, python, CUDA)
- [ ] Add execution utilities in `utils/executor.py` for timeouts, env vars, and safe temp dirs
- [ ] Persist code and run logs under `outputs/{task_id}/runs/{run_id}/` with metadata.json
- [ ] Unit tests: synthesis minimal stub, simulated failure → repair → success, timeout handling

## Dev Notes

- Keep retries targeted: extract error snippet, propose minimal fix, re-run. Avoid oscillations by tracking applied fixes.
- Enforce resource and security constraints (no network calls unless explicitly allowed; respect CUDA_VISIBLE_DEVICES).
- Provide clear return codes and structured errors to the orchestrator for next-step decisions.
