## Story

**As a** researcher,
**I want** an AI agent to generate structured paper cards from collected PDFs that extract key information (methodology, experiments, results, GitHub links) for RAG retrieval,
**so that** the hypothesis generation agent can efficiently retrieve relevant research insights without processing messy unstructured PDFs.

## Background & Context

### What's Been Completed (Story 2.1)
- ✅ **101 papers downloaded** and organized in `.papers/` directory
- ✅ **Paper metadata collected** in `.papers/paper_metadata.json` with title, authors, venue, year, arxiv_id, model_type, attack_level, taxonomy
- ✅ **Categorized by taxonomy**: `any-to-t/` and `any-to-v/` with subdirectories by attack level

### The Problem with Direct PDF Processing
Standard PDF processing tools struggle with academic papers because:
- **Unstructured layout**: Multi-column formats, figures, tables intermixed with text
- **Information overload**: 10-20 pages of dense content, but we only need key sections
- **Poor extraction quality**: Standard tools extract text linearly, losing semantic structure
- **Noise**: References, headers, footers, captions create noisy context for RAG

### Our Solution: AI-Powered Paper Cards
Instead of feeding raw PDFs to RAG, we'll create **structured paper cards** (markdown) that contain:
- **Methodology**: Core method description, key techniques, algorithm approach
- **Experiment Design**: Datasets, models tested, evaluation metrics, baselines
- **Key Results (Summary)**: Main findings, performance highlights, limitations
- **GitHub Links**: Implementation repositories (if available)
- **Relevance Analysis**: How the paper applies to our data-based unlearning research

These cards will be stored in `.paper_cards/{category}` as markdown files, making them:
- **RAG-friendly**: Clean, structured text for embedding and retrieval
- **Human-readable**: Easy to review and verify quality
- **Semantically rich**: Focused on actionable research insights

```python
from pathlib import Path

# Configure Paper Card Extraction Agent

    system_message="You are an expert research paper analyst...",
    model = ModelFactory.create(
      model_platform=ModelPlatformType.OPENAI_COMPATIBLE_MODEL,
      model_type="openai/gpt-5-nano",
      url="https://openrouter.ai/api/v1",
      api_key=os.getenv("OPENROUTER_API_KEY"),
      model_config_dict={"temperature": 0.5},
)

# Extract text from PDF using CAMEL Loaders
def extract_pdf_text(pdf_path: str) -> str:
    """Extract text from PDF using CAMEL's built-in loaders."""
    try:
        # Method 1: UnstructuredIO (preferred for academic papers)
        uio = UnstructuredIO()
        elements = uio.parse_file_or_url(pdf_path)

        # Clean and extract text
        text = "\n\n".join([str(el) for el in elements])

        # Optional: Clean the extracted text
        clean_options = [
            ('replace_unicode_quotes', {}),
            ('clean_dashes', {}),
            ('clean_non_ascii_chars', {}),
            ('clean_extra_whitespace', {}),
        ]
        text = uio.clean_text_data(text=text, clean_options=clean_options)

        return text

    except Exception as e:
        # Fallback: Base IO method
        with open(pdf_path, "rb") as file:
            file_content = file.read()
        file_obj = create_file_from_raw_bytes(file_content, Path(pdf_path).name)

        # Extract text from all pages
        text = "\n\n".join([doc["page_content"] for doc in file_obj.docs])
        return text

# Generate paper card
pdf_text = extract_pdf_text(".papers/any-to-t/input_level/2505.11842.pdf")
card_prompt = f"""Extract structured information from this paper:

{pdf_text}

Generate a paper card with sections: Methodology, Experiments, Results, GitHub links."""

response = extraction_agent.step(card_prompt)
card_content = response.msgs[0].content
```

### CAMEL Loaders for PDF Processing

CAMEL provides multiple loader options for PDF processing (from `external/camel/docs/key_modules/loaders.md`):

**Option 1: UnstructuredIO (Recommended for Academic Papers)**
- Powerful ETL for parsing, cleaning, extracting, chunking unstructured data
- Perfect for RAG pipelines and pre-processing
- Handles multi-column layouts, figures, tables
- Built-in text cleaning (unicode quotes, dashes, extra whitespace)
- Can chunk content by title for better structure

**Option 2: Base IO (Simple Fallback)**
- Basic file I/O for PDF, DOCX, HTML formats
- Returns `File` objects with page content
- Simpler but less sophisticated parsing

**Option 3: Chunkr Reader (Advanced, API-based)**
- Document chunking, segmentation, and OCR
- Handles scanned PDFs
- Returns structured chunks with bounding boxes
- Async API (requires API key)

**Option 4: Mistral Reader (OCR-focused)**
- Mistral AI's OCR service
- Extracts text from images and PDFs
- Supports both local and remote files
- Good for image-heavy papers

**Our Choice: UnstructuredIO with Base IO fallback**
- UnstructuredIO for robust parsing and cleaning
- Base IO as fallback if UnstructuredIO fails
- No external API keys required
- Best balance of quality and simplicity

### GitHub Link Extraction

```python
import re
from typing import Optional

def extract_github_link(pdf_text: str) -> Optional[str]:
    """Extract GitHub repository URL from PDF text."""
    # Regex pattern for GitHub URLs
    github_pattern = r'https?://github\.com/[\w\-]+/[\w\-\.]+'

    matches = re.findall(github_pattern, pdf_text)

    if matches:
        # Return first match (usually main repo)
        return matches[0]

    return None

def validate_github_link(url: str) -> bool:
    """Validate that GitHub URL is accessible."""
    try:
        response = requests.head(url, timeout=5)
        return response.status_code == 200
    except:
        return False
```

### Batch Processing with Progress Tracking

```python
import json
from pathlib import Path
from tqdm import tqdm

def generate_paper_cards_batch(papers_dir: Path, cards_dir: Path):
    """Generate paper cards for all PDFs in papers directory."""

    # Load progress
    progress_file = cards_dir / ".generation_progress.json"
    if progress_file.exists():
        progress = json.loads(progress_file.read_text())
    else:
        progress = {"processed": [], "failed": []}

    # Find all PDFs
    pdf_files = list(papers_dir.rglob("*.pdf"))

    for pdf_path in tqdm(pdf_files, desc="Generating paper cards"):
        arxiv_id = pdf_path.stem

        # Skip if already processed
        if arxiv_id in progress["processed"]:
            continue

        try:
            # Generate card
            card = generate_paper_card(pdf_path)

            # Save card
            card_path = cards_dir / pdf_path.relative_to(papers_dir).with_suffix(".md")
            card_path.parent.mkdir(parents=True, exist_ok=True)
            card_path.write_text(card)

            # Update progress
            progress["processed"].append(arxiv_id)
            progress_file.write_text(json.dumps(progress, indent=2))

        except Exception as e:
            logger.error(f"Failed to process {arxiv_id}: {e}")
            progress["failed"].append({"arxiv_id": arxiv_id, "error": str(e)})
            progress_file.write_text(json.dumps(progress, indent=2))
```

```python
def validate_paper_card(card: PaperCard) -> float:
    """Calculate quality score for extracted paper card."""
    score = 0.0

    # Check required fields are not empty
    if card.core_method and len(card.core_method) > 50:
        score += 0.3
    if len(card.key_techniques) >= 3:
        score += 0.2
    if len(card.datasets_used) >= 1:
        score += 0.2
    if card.main_findings and len(card.main_findings) > 100:
        score += 0.2
    if card.relevance_to_unlearning and len(card.relevance_to_unlearning) > 50:
        score += 0.1

    return score

def retry_on_error(max_retries: int = 3):
    """Retry decorator with exponential backoff."""
    def decorator(func):
        def wrapper(*args, **kwargs):
            delay = 1.0
            for attempt in range(max_retries + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_retries:
                        raise
                    time.sleep(delay)
                    delay *= 2
            raise
        return wrapper
    return decorator
```
